How it works

The Senate accepts the state from a select list and calls the congress API of the sunlight foundation.

The House attempts to use the browser's Geolocation API to get lat/long then calls the Congress API that uses location. If that works it displays it. It also gives the option of using zip code via a less accurate Sunlight foundation API

The historic accepts a state from the list and year, and passes it to my own API which queries a local database. The database contains three tables, representatives, terms and zipcodes. Representative contains information about the representative including the state he/she served in, where terms contains the list of terms that the person served, by start and end date, and a reference back to the representative entry. This data came from a yaml file downloaded from the sunlight foundation website, then parsed using pyyaml and calls to create entries in the database via the django object model. The query then looks for all terms which match the state and where the start date is less than or equal to the year entered, and the end date is greater than or equal to the year entered. We can then loop through the terms returned to find the representative information.

For the state data, to be done later, the Congress API does not support zip codes, so where geolocation api does not work we will need an alternative. In preparation for that I found a csv file which translates zipcodes to an approximate lat/long value. I then use standard python text parsing techniques like split to generate the data to fill the database via the Django Object model.

How the server was built

I am using a separate server and hosted website, as if our server were an independent service vending REST APIS.

The server is a droplet from digital ocean running Ubuntu 14, built using:

  • Python
  • Django
  • Python
  • MySQL

I am hosting our web site on Ecowebhosting

The HTML is being developed using Sublime Text 2 and the python is developed using Springsource version of Eclipse on Windows 7.

I am uploading the http to the website using FireFTP, a plugin inside of Firefox, and using git to control the movement of python between the development Ubuntu VM I have running under VMWare Workstation and the target droplet.
